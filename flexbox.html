TERMS EXPLAINER
Before we continue, it's important that we understand certain terms because in comparing them we might end up a bit confused and also because when we see CSS Grid later we will also see these terms plus a few more which confuse us even further.


"justify":
another way of saying 'main-axis'. It's just a name, it doesn't mean anything in itself.


"align":
another way of saying 'cross-axis'. Again, it's just a name.


"-content":
properties ending with this keyword move the empty container space around the flex/grid items, and thus move the whole block of items/content within the container simultaneously. However, this is as long as the parent container is bigger than its children because empty space is required for these properties to work! That's why it's quite normal to set the parent's height so that it's deliberately bigger than it would normally be, in order to achieve the effects delivered by the '-content' properties. Note: with these properties the items themselves don't change in size (except stretch which makes items take up all free space), only the distribution or positioning of the items relative to parent container (due to the moving round of empty space).


"-items":
properties ending with this keyword look at each column or row track individually and tell the grid items where and how to line up within their individual cells, moving all the items together simultaneously within the track (except stretch which makes items take up all free space). That is, the items line up in relation to each other.


Following this link we can find a great resource which helps us visualise these following properties and see the differences between the different values in real time.


To avoid confusing with CSS Grid, these are the justify and align properties that we can apply to CSS Flexbox:
  • justify-content (aka main-axis-content)


  • align-content (aka cross-axis-content) - Note: requires multiple 'flex-wrap:wrap' rows in order to work.

  • align-items (aka cross-axis-items)

  • align-self (aka cross-axis-item) - Note: is 'align-item' but for one item and this property overrides the more general 'align-items:' declaration.

JUSTIFY-CONTENT (MAIN-AXIS-CONTENT)

The 'justify-content:' property moves the empty container space around the flex items along the main axis, and as a result, moves the whole block of items/content within the container simultaneously. This only works if the parent is bigger than the items (i.e., there's empty space to work with). Here the the items themselves don't change in size (except stretch which makes items take up all free space), only the distribution or positioning of the items relative to parent container (due to the moving round of empty space).

The values we can apply here are: flex-start | flex-end | center | space-between | space-around | space-evenly. Notice the differences in the examples:

flex-start (default): empty space placed to the right, pushing the items to the start of the container's main axis.

Here the the items themselves don't change in size (except stretch which makes items take up all free space), only the distribution or positioning of the items relative to parent container (due to the moving round of empty space).

The 'align-items:' property affects the items along the cross-axis individually, telling the flex items where and how to line up within their individual cells, moving all the items together simultaneously within the track (except stretch which makes items take up all free space). stretch (default): items will stretch across the whole cross axis (normally the whole vertical length).

Item-based Properties


ORDER

We can use the 'order:' property to determine the order the items appear in. By default, flex items will be arranged in the source order with the initial value of order being 0 (zero). If there's a mix of ordered and unordered items, the items without a specific ordering get moved to the front. The problem with this property is that although the visual display order changes, the order in our HTML doesn't change, creating usability problems. It's best to avoid this property if possible!

FLEX-SHRINK

The 'flex-shrink:' property is essentially opposite of the 'flex-grow:' property. If there's not enough space available for the flexbox container items, then this property determines how much a flex item will shrink relative to the rest of the flex items. By default the the value is set to '1' meaning all items will shrink down when the container becomes too small to contain all the items. Setting the value to '0' means the item doesn't shrink nor wrap its content - its sibling items will shrink to give space to the target element - meaning there's a possibility of overflow. Negative numbers are invalid here.

Here we have a container with a max-width of 70vw and items with a width of 20vw. Item 4 has a flex-shrink property of '4' meaning it can shrink x4 more than the other items, meaning no overflow occurs.

FLEX-BASIS

The 'flex-basis:' property determines the initial length/size for an item or items before the empty space is distributed between them. The default value is 'auto' meaning the length is equal to the length of the flexible item (whether that be the content length or a length we determined previously). However, if we set a manually size (e.g. 300px) the item will try to reach that size whenever it can because that is its 'base' size or size of reference. The value we give 'flex-basis:' can be used to decide where we start calculating our 'flex-grow:' value from.

FLEX (SHORTHAND)

The 'flex:' property is a shorthand property to set the 'flex-grow:', 'flex-shrink:' and 'flex-basis:' properties at the same time (in that order) for ONE item. This property is commonly used to create responsive layouts and is recommended over using the individual properties since it automatically defines the other properties if no value for them is given. Remember, each of these properties means:

  • flex-grow: Number specifying how much the item will grow relative to the rest of the flex items (to occupy the empty space).
  • flex-shrink: Number specifying how much the item will shrink relative to the rest of the flex items (if necessary).
  • flex-basis: Determines the initial length/size for an item or items before the empty space is distributed between them.

Here are some different 'flex:' values and their meaning:

  1. '0 1 auto' (default): doesn't allow the item to grow, only shrink

  2. 'auto': the same as writing '1 1 auto' which allows the item to grow as well as shrink, meaning larger items will occupy more space because each item stretches to fit its content.

  3. 'none': the same as writing '0 0 auto' which means the item doesn't grow or shrink.


Here each item has a different 'flex: property' value (try resizing the web browser to notice how they items change or don't change size):

Item #1 = none (no growing or shrinking, stays as wide as the content);
Item #2 = '2 1 auto' (it grows x2 relative to the other items);
Item #3 = auto (grows and shrinks);
Item #4 = '1 1 200px' (grows and shrinks but has a default size of 200px).
